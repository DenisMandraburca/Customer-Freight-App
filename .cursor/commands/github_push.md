# Command: github_push

Purpose: Safely commit all current changes and push the full project to the configured GitHub remote.

---

# Pre-Push Safety Checks (Mandatory)

1. Ensure working directory is not in the middle of a merge or rebase.
2. Ensure no file exceeds 800 lines.
3. Run workspace-aware verification before pushing.

If verification fails → abort push.

---

# Execution Steps

## 1. Detect Current Branch

- Detect the active git branch.
- Abort if branch is detached (HEAD state).

## 2. Stage Changes

- Stage all modified and new files.
- Do NOT include ignored files.

Equivalent command:

`git add -A`

---

## 3. Commit (If Needed)

If there are staged changes:

Commit format:

```
chore(push): checkpoint before github push

- Workspaces affected: <api/web/shared/db>
- Auto-generated by Cursor github_push command
```

If no changes → skip commit.

---

## 4. Pull (Safety Sync)

Run:

`git pull --rebase origin <current-branch>`

If conflicts occur → abort push and report conflicts.

---

## 5. Push to GitHub

Run:

`git push origin <current-branch>`

---

# Required Output

After execution, output:

## Branch

## Commit Hash (if created)

## Push Result

## Remote URL

---

# Failure Conditions (Block Push)

- Verification failed
- Merge conflicts detected
- Detached HEAD
- No remote configured

---

This command ensures safe commit + sync + push in one deterministic flow.
